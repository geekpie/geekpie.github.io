Hexo 使用 Node.js 编译，Hexo 支持 GitHub Flavored Markdown 的所有功能，支持一键部署到 GitHub Pages, Heroku 或其他网站

5.一些关于在 Linux 上部署遇到的问题

在 Linux上遇到了一些问题，cnpm使用官网上的第一种安装方式，没有安装成功。索性不装了，发现npm安装包的时候网速影响也不大。
中间步骤都相同，问题都出在最后向 Github 推送上
配置完，使用推送命令 总是显示错误403
后来在仓库目录下发现了 .deploy_git 隐藏目录，打开发现其逻辑应该是用 hexo 生成了静态文件，然后将静态文件复制到此 git 仓库中，调用 git 命令提交此仓库中的内容。
进入此目录，找到了 .git 目录 ,编辑了 config 配置文件，心想这下从能提交了吧。。。。然而还是403

然后想到了，Hexo 配置文件中也有 GitHub 地址，会不会是这个参数把上面的顶掉了，随即修改了 _config.yml 将 repo 中的链接改成和上述同样的格式，加上账号密码

竟然成功了~！
